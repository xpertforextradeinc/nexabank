## [1.0.0] - 2026-06-29

This represents the first stable production release of the NexaBank digital banking simulation platform. It establishes a complete, full-stack, real-time client-server architecture powered by React, Vite, Tailwind CSS, and Supabase.

### 🚀 Added
*   **Secure Authentication**:
    *   Supabase-backed email/password logins and onboarding.
    *   Third-party Google OAuth sign-in flow.
*   **Role-Based Access Control**:
    *   Administrative dashboard interface completely segregated from user views.
    *   Database-level user promotion caching (`admin_users` relational joins).
*   **Relational Sandbox Database**:
    *   PostgreSQL schema mapping `profiles`, `wallets`, `transactions`, `deposits`, `withdrawals`, `notifications`, `transfer_history`, and `audit_logs` tables.
*   **Transactional SQL Logic**:
    *   Database-level RPC function `transfer_funds()` executing atomic debit/credit updates within safe SQL transactional blocks.
*   **Admin Compliance Tools**:
    *   Interactive interface for approving/rejecting user wire and crypto deposits.
    *   One-click profile freeze, card block, and withdrawal lock controls.
    *   Append-only immutable security activity logging (`audit_logs`).
*   **Bespoke Design System**:
    *   High-contrast, eye-safe slate visual theme using Tailwind CSS and Inter.
    *   Dynamic balance trajectory charts generated by Recharts.
    *   Responsive navigation systems tailored for desktop and mobile breakpoints.
    *   Real-time notifications powered by PostgreSQL database websocket publications.

### 🛡️ Security & Performance Enhancements
*   **Row-Level Security (RLS)**: Activated on all tables, isolating customer database calls to their own user scope.
*   **Multi-Prefix Env Resolution**: Configured Vite bundler config with multiple environment variable prefixes (`VITE_`, `NEXT_PUBLIC_`, `SUPABASE_`) to prevent load-time crashes in diverse cloud environments.
*   **Error Boundaries**: Implemented a global React class error boundary to catch runtime rendering errors and display a clean recovery portal.
